# Browser Breakout Prompt

以下の指示に従って、このディレクトリ内にブラウザで遊べるブロック崩しゲームを実装してください。

## Goal

シンプルだが見た目が整っていて、すぐ遊べるブロック崩しゲームを作成してください。  
成果物はローカルで開いてすぐ遊べる静的ファイル一式にしてください。

## Requirements

- このディレクトリ内だけで完結させる
- HTML / CSS / JavaScript のみで実装する
- 外部ライブラリやビルドツールは使わない
- 最低でも以下のファイルを作る
  - `index.html`
  - `styles.css`
  - `game.js`
- `index.html` をブラウザで開くだけで遊べる

## Gameplay Requirements

- パドルを左右に動かしてボールを打ち返す
- ブロックをすべて壊すとクリア
- ボールを落とすとライフが減る
- ライフが 0 になったらゲームオーバー
- スコア表示を入れる
- 残りライフ表示を入れる
- クリア時とゲームオーバー時に再スタートできる

## Controls

- キーボード操作に対応する
  - `ArrowLeft` / `ArrowRight`
  - できれば `A` / `D` にも対応
- 可能ならマウス操作にも対応する

## UX / Visual Direction

- 見た目はチープすぎないようにする
- 背景、パドル、ボール、ブロックにきちんと色とコントラストを付ける
- 画面上部にタイトル、スコア、ライフを見やすく配置する
- PC ブラウザで快適に遊べるレイアウトにする
- 可能なら軽いアニメーションやヒット時の演出を入れる

## Technical Expectations

- `canvas` を使ってよい
- 衝突判定は安定して動くようにする
- リロードなしで再スタートできるようにする
- コードは読みやすく整理する
- 複雑な箇所にだけ短いコメントを入れる

## Nice To Have

- 難易度が少しずつ上がる
- ブロックの色分け
- スタート画面
- 一時停止
- 効果音なしでも気持ちよく遊べる手触り

## Output Format

作業後は以下を簡潔に報告してください。

1. 作成・更新したファイル
2. 実装した機能
3. ブラウザでの起動方法
4. 既知の制限があればその内容

## Important

- 説明だけで終わらず、実際にファイルを作成すること
- 既存ファイルがあれば尊重しつつ、必要な変更を加えること
- このディレクトリ外には原則として書き込まないこと
